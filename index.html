Что такое HTML и как он работает

HTML (HyperText Markup Language) — это язык разметки для создания структуры веб-страниц. Он описывает что находится на странице: заголовки, абзацы, изображения, ссылки, формы. За внешний вид отвечает CSS, за интерактивность — JavaScript.

Браузер читает HTML и строит DOM-дерево (Document Object Model) — иерархическое представление элементов. CSS «подкрашивает» эти элементы, а JS позволяет изменять DOM и реагировать на действия пользователя.

Важно: Старайтесь писать семантический HTML — использовать элементы по назначению. Это улучшает доступность, SEO и поддержку кода.

  4. Ссылки, якоря и навигация.

  7. Таблицы: базовая разметка и доступность.
  8. Формы: поля ввода, элементы управления, валидация.
  9. Атрибуты: глобаль��ые и специфичные; data-*; ARIA-атрибуты.
  10. Метаданные, SEO-минимум и социальные превью.

Структура HTML-документа

<!doctype html>
<html lang="ru">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Заголовок страницы</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <h1>Привет, HTML!</h1>
  </body>
</html>
  • <!doctype html> сообщает браузеру современный стандарт.
  • lang помогает поиску и доступности (выбор языка).
  • meta viewport делает страницу адаптивной на мобильных.

Текст и заголовки

Заголовки: h1–h6. Основной заголовок на странице — обычно h1. Абзацы — p. Выделения — strong, em. Горизонтальная линия — hr. Перенос строки — br (используйте умеренно). 

<h1>Заголовок страницы</h1>
<p>Обычный абзац текста с <strong>важным</strong> словом и <em>акцентом</em>.</p>
<ul>
  <li>Пункт списка</li>
  <li>Ещё пункт</li>
</ul>

Ссылки и навигация

Ссылка — элемент a с атрибутом href. Для открытия в новой вкладке добавьте target="_blank" и ради безопасности rel="noopener noreferrer". 

<nav>
  <a href="#top">К началу</a>
  <a href="https://developer.mozilla.org/ru/" target="_blank" rel="noopener noreferrer">MDN</a>
</nav>

Изображения и медиа

Используйте осмысленный alt для изображений. Для адаптивной графики применяйте <picture>, srcset и sizes. 

<img src="image.jpg" alt="Описание изображения" width="600" height="400" />

<picture>
  <source srcset="image@2x.webp 2x, image.webp 1x" type="image/webp" />
  <img src="image.jpg" alt="Описание" />
</picture>

<video controls width="480">
  <source src="video.mp4" type="video/mp4" />
  Ваш браузер не поддерживает видео.
</video>

Семантические теги

Семантика делает документ понятнее для людей, машин и вспомогательных технологий.

<header>Шапка</header>
<nav>Навигация</nav>
<main>
  <section>
    <article>Статья</article>
  </section>
  <aside>Боковая панель</aside>
</main>
<footer>Подвал</footer>

Таблицы

<table>
  <caption>Продажи за месяц</caption>
  <thead>
    <tr><th scope="col">Товар</th><th scope="col">Кол-во</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">Яблоки</th><td>15</td></tr>
    <tr><th scope="row">Груши</th><td>10</td></tr>
  </tbody>
</table>

Добавляйте caption и scope для лучшей доступности.

Формы

<form action="/search" method="get">
  <label for="q">Поиск</label>
  <input id="q" name="q" type="search" placeholder="Введите запрос" required />
  <button type="submit">Искать</button>
</form>

<form action="/register" method="post">
  <fieldset>
    <legend>Регистрация</legend>
    <label>Email
      <input type="email" name="email" required />
    </label>
    <label>Пароль
      <input type="password" name="password" minlength="8" required />
    </label>
    <label>Возраст
      <input type="number" name="age" min="1" max="120" />
    </label>
    <label>Страна
      <select name="country" required>
        <option value="" disabled selected>Выберите</option>
        <option>Россия</option>
        <option>Казахстан</option>
      </select>
    </label>
  </fieldset>
  <button type="submit">Создать аккаунт</button>
</form>

Соотносите label и поле через for/id (или вкладывайте поле внутрь label) — это улучшает доступность и UX.

Атрибуты и глобальные атрибуты

  • Глобальные: id, class, style, title, hidden, tabindex, data-*, aria-*.
  • Спе��ифичные: например, href у a, src у img, type у input. 
<button class="btn" data-track="cta" aria-label="Открыть форму">Открыть</button>

Метаданные, SEO и соц-превью

<meta name="description" content="Краткое описание страницы" />
<meta property="og:title" content="Заголовок для соцсетей" />
<meta property="og:description" content="Описание для соцсетей" />
<meta property="og:image" content="https://example.com/preview.jpg" />
<link rel="icon" href="/favicon.ico" />

Доступность (A11y)

  • У изображений есть осмысленные alt.
  • Ссылки содер��ат понятный текст (не «кликни сюда»).
  • Формы связаны с label, есть подсказки об ошибках.
  • Контент доступен с клавиатуры, есть видимый фокус.
  • Семантическая разметка вместо сплошных div.

Структура проекта и файлы

project/
├─ index.html
├─ index.css
├─ images/
└─ scripts/

Используйте относительные пути: images/logo.png. Не кладите тяжёлые медиа в корень.

Как работать с HTML локально

  1. Откройте файл index.html двойным кликом или через браузер (Файл → Открыть).
  2. В редакторе (VS Code) установите расширение «Live Server» для автоматического обновления при сохранении.
  3. Проверяйте валидность на validator.w3.org — вставьте HTML-код или укажите URL.

Практика и упражнения

  1. Сверстайте визитку: заголовок, абзац, список ссылок, картинка.
  2. Добавьте форму обратной связи: имя, email, сообщение.
  3. Сделайте навигацию по секциям внутри страницы через якоря.
  4. Разметьте статью с использованием article и section.

Частые ошибки

  • Отсутствие <meta name="viewport"> — страница не адаптивна.
  • Неверные вложенности тегов (например, li вне ul/ol).
  • Отсутствие alt у изображений.
  • Использование таблиц для вёрстки вместо CSS.
  • Длинные «простыни» без заголовков и секций.

Вывод

HTML — фундамент веба. Освоив структуру документа, семантику и основы доступности, вы сможете уверенно создавать понятные и устойчивые страницы. Закрепляйте знания практикой: маленькие проекты, чтение документации и разбор чужого кода дадут быстрый прогресс.
